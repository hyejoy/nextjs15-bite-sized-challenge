📝 section02-02 Next.js 사전렌더링 이해하기
 
 🔥 용어 정리
 * 브라우저 ≒ 클라이언트
 * FCP (First Contentful Pain) ≒ 요청 시작 시점 ~ 컨텐츠 렌더링 되기까지의 시간
 * 랜더링 1) 서버에서 JS 실행 ≒ React 컴포넌트를 HTML로 변환하는 과정
		 2) HTML코드를 브라우저가 화면에 그려내는 작업
 * TTI (Time to Interactive) ≒ 요청 시작 시점 ~ 수화 되기까지의 시간
 
 ✅ 사전 렌더링 이해하기
 
 - CSR 단점인 초기접속 요청 속도 (FCP) 향상, CSR 장점인 빠른 페이지 이동의 결합 형태
 - 브라우저의 요청에 사전에 렌더링이 완료된 HTML을 응답하는 렌더링 방식
 - CSR(Client Side Rendering)의 단점을 효율적으로 해결하는 방식
 - 사전 렌더링은 유저가 브라우저를 통해 접속 요청을 하면 서버에서 JS실행 (렌더링)하여
   렌더링된 HTML을 브라우저로 전송해주는 방식이야
  
 ✅ CSR 이해하기
 
 - React 기본적인 렌더링 방식이야
 - CSR에서는 유저가 브라우저에게 접속 요청을 하게되면 
   서버가 브라우저에게 index.html(빈껍데기)와 JS파일을 번들링해 후속으로 브라우저에게 보내주게 돼 
 - 브라우저에서는 후속으로 받은 번들링된 JS파일을 실행해
 - JS파일 실행이 되면 유저에게 컨테츠 렌더링을 시켜
 - 유저는 이제서야 요청한 페이지를 보게 돼
 
  ✅ CSR 장점
 
 - 장점 : 초기 접속 이후 페이지 이동을 빠른 속도로 전환할 수 있어
 - 단점 : 초기 접속이 느려짐 ≒ FCP 느려짐
		 └── 브라우저가 화면의 컨텐츠를 렌더링 하기전까지, index.html과
			 JS Bundle파일도 받아와야하고, 실행까지 마쳐야하기때문에 이 과정이 오래걸림
 
 ✅ Next.js 사용으로 인해 CSR의 문제점 해결 
 
 - 유저가 브라우저로 접속 요청을 하게 되면, 서버에서 JS실행(렌더링)까지 마치는 과정을 거친다.
   서버가 렌더링 된 HTML을 브라우저에게 전달해주고,
   이 HTML을 그대로 화면에 렌더링시켜 ≒ FCP 빨라지
   화면에 렌더링 된 직후 자바스크립트가 처리하는 영역인 인터렉션(상호작용)은 아직 불가능해
   서버가 후속으로 JS 번들링된 결과를 브라우저에게 보내주면
   서버로부터 받은 JS 코드를 직접 실행하여, 화면에 렌더링 되어있는 HTML 요소들과 연결 시켜주게 돼 (수화)
   수화과정까지 거치게 되면 웹 브라우저가 상호작용이 가능해져
   
 ✅ 초기 접속 요청 이후에 페이지 이동은 어떻게 할까?
 
 - CSR과 동일한 방식으로 브라우저에서 JS 코드를 실행해 컴퍼넌트를 교체하는 방식으로 페이지 이동하게 됨
